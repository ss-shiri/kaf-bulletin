# KAF BULLETIN

بولتن رصد منابع باز شیمیایی و بیولوژیک در دوازده زبان.

هر خبر به همان زبانی که منتشر شده نمایش داده می‌شود، با منبعش زیر آن. بدون ترجمه، بدون تأیید، بدون رتبه‌بندی، بدون بازنویسی. تنها ادعای این بولتن، منبع است.

**زبان‌ها:** انگلیسی، فرانسوی، اسپانیایی، عربی، روسی، چینی، آلمانی، عبری، هلندی، ژاپنی، کره‌ای، فارسی

## راه‌اندازی، قدم به قدم

### گام ۱ — repo را بسازید

فایل‌ها را در یک پوشه بگذارید و:

```bash
cd kaf-bulletin
git init
git add .
git commit -m "initial"
git branch -M main
```

بعد در github.com یک repository خالی به نام `kaf-bulletin` بسازید (public) و:

```bash
git remote add origin https://github.com/ss-shiri/kaf-bulletin.git
git push -u origin main
```

اگر `gh` نصب دارید، هر سه خط بالا با یک دستور انجام می‌شود:

```bash
gh repo create kaf-bulletin --public --source=. --push
```

### گام ۲ — اجازه‌ی نوشتن به Actions بدهید

**Settings ← Actions ← General ← Workflow permissions**

گزینه‌ی **Read and write permissions** را انتخاب و Save کنید.

بدون این کار، مرحله‌ی commit شکست می‌خورد و آرشیو ذخیره نمی‌شود.

### گام ۳ — GitHub Pages را روشن کنید

**Settings ← Pages ← Source**

گزینه‌ی **GitHub Actions** را انتخاب کنید. (نه «Deploy from a branch»)

### گام ۴ — اولین چاپ

**Actions ← update ← Run workflow ← Run workflow**

حدود پنج تا هشت دقیقه طول می‌کشد، چون بین درخواست‌ها مکث می‌گذارد تا خزنده‌ی مؤدبی باشد.

بعد از اتمام، سایت اینجاست:

```
https://ss-shiri.github.io/kaf-bulletin/
```

### گام ۵ — رها کنید

از این به بعد خودش هر شش ساعت اجرا می‌شود. کاری لازم نیست.

## چطور تنظیمش کنم

همه چیز در یک فایل است: `config/lexicon.json`

ساختار هر زبان:

```json
"fa": {
  "name": "Persian",
  "native": "فارسی",
  "dir": "rtl",
  "gnews": {"hl": "fa", "gl": "IR", "ceid": "IR:fa"},
  "chemical": {
    "query": ["سلاح شیمیایی", "گاز کلر", "..."],
    "match": ["سلاح شیمیایی", "سلاح‌های شیمیایی", "..."]
  },
  "biological": { "query": [...], "match": [...] }
}
```

تفاوت `query` و `match` مهم است:

| میدان | کارکرد |
| --- | --- |
| `query` | این‌ها به سرچ ارسال می‌شوند. هرچه بیشتر، درخواست بیشتر و اجرا کندتر. حدود ۸ تا کافی است |
| `match` | این‌ها روی تیتر برگشتی تطبیق داده می‌شوند تا موضوع تعیین شود. هرچه بیشتر، بهتر. هزینه‌ای ندارد |

**برای اضافه کردن اصطلاح جدید** فقط داخل `match` بگذارید. **برای گسترش دامنه‌ی جستجو** به `query` هم اضافه کنید.

**برای حذف یک زبان** کل بلوک آن را پاک کنید. **برای افزودن زبان جدید** یک بلوک مشابه بسازید. کد سایت و خزنده هر دو خودکار با آن تطبیق می‌یابند.

تنظیمات دیگر در انتهای همان فایل:

```json
"settings": {
  "chunk_size": 8,               تعداد اصطلاح در هر درخواست
  "request_delay_seconds": 1.2,  مکث بین درخواست‌ها
  "latest_window_days": 10,      چند روز اخیر در صفحه دیده شود
  "max_items_in_latest": 1200,   سقف آیتم روی صفحه
  "seen_retention_days": 120     حافظه‌ی تکراری‌زدایی
}
```

## نکته‌ای که باید بدانید: «هر یک دقیقه» ممکن نیست

گیت‌هاب حداقل بازه‌ی اجرای زمان‌بندی‌شده را روی پنج دقیقه گذاشته و هر چیزی سریع‌تر را **بی‌صدا** نادیده می‌گیرد. یعنی YAML شما درست پارس می‌شود، زمان‌بندی در صفحه نمایش داده می‌شود، و هرگز اجرا نمی‌شود.

حتی روی پنج دقیقه هم تأخیر پنج تا سی دقیقه معمول است و در ساعات شلوغ ممکن است اجرا کاملاً حذف شود.

دلیل دوم: خزیدن هر دقیقه روی گوگل‌نیوز به بلاک شدن ختم می‌شود، نه به داده‌ی بیشتر.

**هر شش ساعت تنظیم شده است.** اگر تندتر می‌خواهید، در `.github/workflows/update.yml` این خط را عوض کنید:

```yaml
- cron: "17 */6 * * *"     # هر ۶ ساعت (پیش‌فرض)
- cron: "17 */3 * * *"     # هر ۳ ساعت
- cron: "17 * * * *"       # هر ۱ ساعت (حداقل معقول)
```

## اجرای محلی

```bash
python3 scripts/selftest.py    # ۳۹ تست، بدون شبکه
python3 scripts/ingest.py      # خزیدن واقعی
python3 -m http.server 8000    # بعد باز کنید: localhost:8000
```

تست‌ها روی یک پوشه‌ی موقت اجرا می‌شوند و به آرشیو واقعی دست نمی‌زنند.

```
data/index.json         فهرست آرشیو

```

### آرشیو فقط‌افزودنی

هیچ رکوردی بازنویسی نمی‌شود. هر تیتر زمان **اولین مشاهده** را برای همیشه نگه می‌دارد، حتی اگر در اجرای بعدی دوباره ظاهر شود. تاریخچه‌ی git زیر همه‌ی این‌ها لایه‌ی تغییرناپذیر است.

این تنها دارایی این پروژه است که با گذر زمان ارزشش بالا می‌رود و هیچ‌کس نمی‌تواند بعداً بازتولیدش کند. برای همین از روز اول ثبت می‌شود.

## برچسب موضوع چطور تعیین می‌شود

برچسب از روی **خود تیتر** تعیین می‌شود، نه از روی اینکه کدام کوئری آن را برگردانده.

اگر تیتر اصطلاح شیمیایی داشته باشد ← `CHEM`
اگر اصطلاح بیولوژیک داشته باشد ← `BIO`
اگر هر دو ← هر دو برچسب
اگر هیچ‌کدام (اصطلاح احتمالاً در متن مقاله است) ← برچسب کوئری، با نشان `UNCONF`

نسخه‌ی اول کد برچسب را از کوئری می‌گرفت و نتیجه این بود که تقریباً همه چیز هر دو برچسب را می‌گرفت و فیلتر بی‌فایده می‌شد. تست این را گرفت و اصلاح شد.
